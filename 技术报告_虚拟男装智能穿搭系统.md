## 1. 项目概述与技术架构

### 1.1 项目背景与目标

本项目旨在为男性用户提供一款智能化的虚拟穿搭推荐与试穿系统。用户可以通过上传自己的衣物照片，系统基于AI技术自动生成穿搭搭配建议，并支持虚拟试穿功能，让用户在不出门的情况下预览不同衣物的搭配效果。

**核心价值**：
- 解决男性用户"不知道怎么搭配"的痛点
- 降低购物决策成本，提升购物体验
- 利用AI技术实现个性化的智能穿搭推荐

### 1.2 系统架构设计

## 2. 核心功能模块实现

### 2.1 虚拟试衣模块（Virtual Try-On）

#### 2.1.1 功能描述

用户上传人物照片和衣物照片，系统通过AI模型生成穿着该衣物的效果图。

#### 2.1.2 核心实现

**API调用流程**：

```
用户上传图片
     │
     ▼
图片压缩与预处理（sharp）─► 上传到OSS ─► 获取公共URL
     │
     ▼
调用 DashScope ai_tryon-plus API（异步任务模式）
     │
     ▼
轮询任务状态（最多60次，间隔2秒）
     │
     ├── SUCCEEDED ─► 下载结果图片 ─► 转存到OSS ─► 返回公共URL
     │
     ├── FAILED ─► 记录错误日志 ─► 返回失败信息
     │
     └── 超时（120秒）─► 返回超时错误
```

**关键代码实现**：

```python
# dashscope_tryon 函数核心逻辑
def dashscope_tryon(person_url: str, garment_url: str, garment_type: str) -> Dict[str, Any]:
    # 1. 检查缓存
    cache_key = _get_tryon_cache_key(person_url, garment_url, garment_type)
    with _tryon_cache_lock:
        cached = _tryon_cache.get(cache_key)
        if cached and (time.time() - cached.get("ts", 0)) < _tryon_cache_ttl:
            return cached["result"]  # 直接返回缓存结果

    # 2. 构建API请求
    headers = {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "X-DashScope-Async": "enable"  # 启用异步模式
    }

    payload = {
        "model": "ai_tryon-plus",
        "input": {
            "person_url": person_url,
            "garment_url": garment_url,
            "garment_type": garment_type
        },
        "parameters": {"resolution": -1}
    }

    # 3. 创建异步任务
    resp = requests.post(dashscope_api_url, headers=headers, json=payload, timeout=30)
    task_id = resp.json().get("output", {}).get("task_id")

    # 4. 轮询任务结果
    for _ in range(60):  # 最多60次
        time.sleep(2)  # 每次间隔2秒
        status_resp = requests.get(f"{dashscope_task_url}/{task_id}", headers=headers)
        task_status = status_resp.json().get("output", {}).get("task_status")

        if task_status == "SUCCEEDED":
            image_url = status_resp.json().get("output", {}).get("results", [{}])[0].get("url")
            # 转存到OSS并返回公共URL
            return _store_tryon_result(image_url)
        elif task_status == "FAILED":
            return {"success": False, "message": "AI生成失败"}

    return {"success": False, "message": "任务超时"}
```

#### 2.1.3 本地降级方案

当DashScope API不可用时，系统提供基于PIL的本地图像合成方案作为保底：

```python
def _simple_tryon_compose(person_bytes: bytes, garment_bytes: bytes, garment_type: str) -> bytes:
    """
    本地图像合成算法
    - 将衣物图片缩放后叠加到人物图片指定位置
    - 使用alpha通道实现透明融合
    - 自动去除白色背景（纯色衣物）
    """
    p = Image.open(io.BytesIO(person_bytes)).convert("RGBA")
    g = Image.open(io.BytesIO(garment_bytes)).convert("RGBA")

    # 根据衣物类型计算缩放比例和位置
    if garment_type == "bottom":
        scale = 0.62  # 裤子需要覆盖下半身
        x0 = int(pw * 0.19)
        y0 = int(ph * 0.52)
    else:  # top, dress
        scale = 0.68  # 上衣覆盖上半身
        x0 = int(pw * 0.16)
        y0 = int(ph * 0.20)

    # 等比例缩放
    target_w = max(80, int(pw * scale))
    g = g.resize((target_w, target_h), Image.LANCZOS)

    # 叠加融合
    canvas.alpha_composite(Image.new("RGBA", (pw, ph), (0, 0, 0, 0)))
    layer = Image.new("RGBA", (pw, ph), (0, 0, 0, 0))
    layer.paste(g, (x0, y0), mask=alpha)  # 使用alpha通道作为mask
    out = Image.alpha_composite(canvas, layer).convert("RGB")

    buf = io.BytesIO()
    out.save(buf, format="JPEG", quality=85, dpi=(72, 72))
    return buf.getvalue()
```

### 2.2 智能穿搭推荐模块

#### 2.2.1 功能描述

根据用户衣橱中的衣物、当前天气、场景和风格偏好，生成个性化的搭配方案。

#### 2.2.2 推荐算法设计

**输入参数**：
- 用户衣物列表（从数据库获取）
- 场景（商务会议、休闲聚会、户外运动等）
- 目的（日常穿着、正式场合、约会等）
- 风格偏好（商务正式、休闲时尚、运动风格等）
- 天气信息（温度、天气类型）
- 当前温度（用于筛选适合的衣物）

**推荐策略**：

```python
def outfit_recommend(user_id: int, scenario: str, purpose: str, style: str,
                    weather_type: str, tempC: float) -> Dict[str, Any]:
    # 1. 获取用户衣物
    user_clothes = fetch_user_clothes_from_db(user_id)

    # 2. 按衣物类型分组
    by_type = {"上衣": [], "裤子": [], "外套": [], "鞋子": [], "配饰": []}
    for item in user_clothes:
        item_type = item.get("type", "")
        if item_type in by_type:
            by_type[item_type].append(item)

    # 3. 根据温度筛选适合的衣物
    def is_suitable_for_temp(item, temp):
        suitable = item.get("suitable_temp", "")
        if not suitable:
            return True
        # 解析温度范围，如 "10-20" 表示10到20度
        try:
            if "-" in suitable:
                min_t, max_t = map(float, suitable.split("-"))
                return min_t <= temp <= max_t
        except:
            pass
        return True

    # 4. 根据场景和风格生成搭配
    for _ in range(3):  # 生成3套方案
        selected = {}
        # 优先选择与场景风格匹配的衣物
        for item_type, items in by_type.items():
            matched = [i for i in items if is_suitable_for_temp(i, tempC)]
            if matched:
                # 随机选择一件匹配的衣物
                selected[item_type] = random.choice(matched)

        # 5. 分析缺失单品
        missing_items = []
        for required_type in ["上衣", "裤子", "鞋子"]:
            if required_type not in selected:
                missing_items.append(required_type)

        # 6. 为缺失单品生成AI图片
        for missing_type in missing_items:
            prompt = f"男士{missing_type}商品实拍图，白底，高清8K"
            generated = text_to_img(prompt, hint_key=style)
            if generated:
                missing_items_info.append({
                    "type": missing_type,
                    "generated_url": generated
                })

    return {
        "success": True,
        "data": {
            "outfits": outfits,  # 3套搭配方案
            "missing_items_info": missing_items_info  # 缺失单品信息
        }
    }
```

### 2.3 天气查询模块

#### 2.3.1 功能描述

#### 2.3.2 实现机制

```python
def fetch_weather(lat: float, lon: float, unit: str = "C") -> Dict[str, Any]:
    """
    天气查询，支持摄氏/华氏温度单位
    返回：温度、体感温度、湿度、风速、天气类型
    """
    # 生成缓存键：保留3位小数平衡精度与缓存命中率
    cache_key = f"geo:{unit}:{round(lat, 3)}:{round(lon, 3)}"

    # 缓存检查（10分钟TTL）
    with _weather_cache_lock:
        cached = _weather_cache.get(cache_key)
        if cached and (time.time() - cached["ts"]) < _WEATHER_CACHE_TTL:
            return cached["data"]  # 返回缓存数据

    # 调用高德天气API
    api_url = f"https://restapi.amap.com/v3/weather/weatherInfo"
    params = {
        "key": AMAP_WEATHER_KEY,
        "location": f"{lon},{lat}",  # 高德API使用 经度,纬度 格式
        "extensions": "base"  # base:基本天气 all:完整天气
    }

    resp = requests.get(api_url, params=params, timeout=10)
    data = resp.json()

    # 解析天气数据
    weather = data.get("lives", [{}])[0] or {}
    result = {
        "temperature": float(weather.get("temperature", 0)),
        "feels_like": float(weather.get("temperature_float", 0)),
        "humidity": float(weather.get("humidity", 0)),
        "wind_speed": float(weather.get("windSpeed", 0)),
        "weather_type": weather.get("weather", ""),
        "unit": unit
    }

    # 写入缓存
    with _weather_cache_lock:
        _weather_cache[cache_key] = {"ts": time.time(), "data": result}

    return result
```

#### 3.1.2 缓存键设计

**试衣缓存键**：
```python
def _get_tryon_cache_key(person_url: str, garment_url: str, garment_type: str) -> str:
    """使用MD5哈希确保缓存键紧凑且唯一"""
    key = f"{person_url}|{garment_url}|{garment_type}"
    return hashlib.md5(key.encode()).hexdigest()
# 示例：f"https://example.com/person.jpg|https://example.com/shirt.jpg|top"
```

```python
    """
    上传到OSS并设置公共读权限
    """
    headers = {
        "Content-Type": content_type,
        "x-oss-object-acl": "public-read"  # 永久公共读权限
    }

    bucket.put_object(object_key, file_bytes, headers=headers)

    # 构建公共读URL（不含签名）
    endpoint = oss_config["endpoint"].replace("https://", "").replace("http://", "")
    public_url = f"https://{bucket.bucket_name}.{endpoint}/{object_key}"

    return {"image_url": public_url, "image_key": object_key}
```

### 5.3 缓存一致性问题

**问题场景**：在高并发环境下，可能出现缓存数据与数据库不一致的情况。

1. **写操作时删除缓存**：更新用户数据时，主动删除相关缓存
2. **设置合理TTL**：避免缓存过期时间过长导致数据陈旧
3. **使用版本号**：在缓存键中包含数据版本号

```python
    """
    验证试衣图片URL的有效性
    - 必须是http或https协议
    - 不能是内网地址（如127.0.0.1、localhost）
    - 不能包含可疑的认证参数
    """
    try:
        p = urlparse(url)
        if not p.scheme.startswith("http"):
            return False

        host = (p.hostname or "").lower()
        # 禁止内网地址
        if host.startswith("127.") or host.startswith("0.") or host == "::1":
```

#### R6: 高并发导致线程阻塞

**风险分析**：
- Python端使用ThreadPoolExecutor处理AI任务
- 线程池满时，新任务需排队等待

**应对措施**：
```python
# 任务队列和线程池配置
_task_queue = deque(maxlen=1000)  # 队列最大1000
_task_executor = ThreadPoolExecutor(max_workers=5)  # 最多5个并发AI任务

def submit_tryon_task(person_url: str, garment_url: str, garment_type: str) -> str:
    """
    提交试衣任务到队列
    返回任务ID用于查询结果
    """
    task_id = str(uuid.uuid4())

    # 队列满时拒绝新任务
    if len(_task_queue) >= 1000:
        raise Exception("系统繁忙，请稍后再试")

    # 异步提交任务
    future = _task_executor.submit(_process_tryon_task, task_id, person_url, garment_url, garment_type)
    _task_queue.append({"task_id": task_id, "future": future})

    return task_id

def get_task_status(task_id: str) -> Dict[str, Any]:
    """
    查询任务状态
    """
    result = _task_results.get(task_id)
    if result:
        return result

    # 检查任务是否还在队列中
    for item in _task_queue:
        if item["task_id"] == task_id:
            return {"status": "pending", "message": "任务处理中"}

    return {"status": "not_found"}
```
